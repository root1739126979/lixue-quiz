# 砺学多题库刷题平台设计说明

> **文档状态：已过期，仅作为备份保留。**
>
> 本文档是早期设计版本，后续实现与评审请以 `docs/superpowers/specs/2026-05-28-lixue-quiz-platform-mvp-v2.md` 为准。该新版已补充 MVP 范围分层、导入预检、账号策略、积分口径、考试快照、AI 缓存失效、安全要求和验收标准。

## 背景

项目书《「砺学」AI知识教育刷题系统项目申报书》提出建设飞书内 H5 刷题工具，用于员工碎片化练习、错题巩固、AI 讲解、积分排名和培训数据汇总。结合当前已准备的 `question/questions.csv` 题库内容，第一版产品定位调整为“砺学多题库学习刷题平台”：先上线设备/岗位知识题库，后续扩展 AI 考试题库、安全题库、考证题库等。

## 目标

MVP 在 1-2 个月内完成可真实内测的版本，部署到云服务器，支撑员工登录、多题库选择、练习答题、错题库、排行榜、轻量模拟考试、按需 AI 讲解、管理后台导入题库、数据看板和基础导出。

MVP 不实现 LLM 自动出题、飞书登录、飞书通知、多角色后台、复杂运营任务和多基地权限。这些能力放入二期或三期。

## 产品定位

砺学是面向基地员工的多题库刷题平台。系统第一批题库为设备/岗位知识题库，同时保留 AI 知识题库扩展能力。员工可在移动端完成题库选择、随机练习、错题重练、模拟考试和排行榜查看。管理员在 Web 后台维护员工、题库、题目、积分规则、LLM 配置，并查看学习数据。

## 技术架构

- 前端：React + Vite，包含员工 H5 和管理后台。
- 后端：FastAPI，提供用户、题库、答题、积分、考试、排行榜、AI 讲解、数据导出等 API。
- 数据库：PostgreSQL。
- 部署：云服务器 + Nginx + HTTPS 域名。
- LLM：OpenAI-compatible 配置，支持 `base_url`、`api_key`、`model`。
- 飞书：MVP 预留接入字段和登录入口，不作为第一版强依赖。

## 用户与权限

MVP 使用后台导入员工表的方式维护用户。员工使用工号或手机号登录。用户记录包含姓名、工号、手机号、部门、状态和预留飞书用户 ID。

后台第一版只有管理员一种角色。管理员可使用全部后台功能，包括员工导入、题库导入、题目编辑、积分规则配置、LLM 配置、数据看板和报表导出。

## 员工端功能

### 题库选择

员工登录后进入题库选择页，查看可用题库，例如“设备知识题库”“AI 知识题库”。每个题库显示题目数、已练习数、正确率和个人积分。

### 练习答题

员工选择题库后可开始练习。MVP 支持随机练习和按题库练习。题目以单选题为主，兼容选项 A-E。答题后立即显示正确答案、CSV 原解析，并记录答题结果。

### 错题库

答错题目自动进入个人错题库。员工可按题库查看错题，重新作答。错题重做正确后，系统将该题标记为已掌握，并从未掌握错题列表中移除。

### AI 讲解

答题反馈页和错题详情页展示 CSV 原解析。员工点击“AI 讲解”时，后端调用 OpenAI-compatible LLM 生成补充讲解，包括正确答案说明、易错点和关联知识提示。生成结果写入缓存，同一道题后续复用缓存内容。

### 排行榜

MVP 提供全局总榜和题库内榜单。全局总榜按所有题库积分合计排序。题库内榜单按指定题库积分排序。

### 模拟考试

MVP 提供轻量模拟考试。管理员可为题库配置固定题量和限时。员工开始考试后，系统随机抽题，交卷后展示得分、正确率和错题明细，并保存考试成绩。

### 个人中心

个人中心展示姓名、部门、总积分、累计答题数、正确率、错题数量和考试记录入口。

## 管理后台功能

### 管理员登录

后台使用管理员账号密码登录。MVP 不做多角色权限。

### 员工导入

管理员通过 CSV 或 Excel 导入员工信息。系统校验工号或手机号唯一性，并生成可登录用户。导入字段包含姓名、工号、手机号、部门和状态。

### 题库管理

管理员可创建、编辑、启用和停用题库。题库字段包含名称、说明、状态、默认考试题量、默认考试限时。

### 题目导入

管理员选择目标题库后导入 CSV 题目。MVP 支持当前 `question/questions.csv` 字段：

- 题号
- 题型
- 题干
- 选项A
- 选项B
- 选项C
- 选项D
- 选项E
- 正确答案
- 正确答案文本
- 解析

当前 CSV 不包含知识模块和难度。MVP 导入时将知识模块默认设为“未分类”，难度默认设为“普通”。后续可通过后台编辑或二期批量补充。

### 题目维护

管理员可查看、搜索、编辑、启用和停用题目。题目停用后不再进入练习和考试抽题。

### 积分规则配置

后台支持配置答题基础分、答对额外分、完成模拟考试分和每日积分上限。积分规则变更后只影响后续行为，不回算历史积分。

### 数据看板

后台看板展示参与人数、活跃人数、累计答题数、整体正确率、题库正确率、错题高频题、模拟考试成绩分布和排行榜概览。

### 报表导出

后台支持导出用户学习汇总、答题记录和模拟考试成绩。导出格式为 CSV 或 Excel。

### LLM 配置

后台支持配置 OpenAI-compatible LLM 参数，包括 `base_url`、`api_key`、`model` 和启用状态。未启用 LLM 时，员工端隐藏或禁用“AI 讲解”入口。

## 数据模型

核心数据表包括：

- users：员工用户。
- admins：后台管理员。
- question_banks：题库。
- questions：题目。
- question_options：题目选项。
- practice_sessions：练习会话。
- answer_records：答题记录。
- wrong_questions：个人错题状态。
- point_rules：积分规则。
- point_transactions：积分流水。
- quiz_scores：按题库汇总积分和正确率。
- exams：模拟考试配置。
- exam_attempts：考试记录。
- exam_answers：考试答题明细。
- ai_explanations：AI 讲解缓存。
- llm_configs：LLM 配置。

## 分阶段路线

### MVP

完成多题库、CSV 题库导入、员工登录、练习答题、错题库、全局和题库排行榜、轻量模拟考试、按需 AI 讲解、后台数据看板和基础导出。

### 二期

接入飞书登录、飞书通知、AI 出题审核池、知识模块管理、薄弱点推荐、模拟考试策略增强和学习提醒。

### 三期

支持多基地和多部门权限、部门榜、更多题库类型、运营活动、案例集材料沉淀报表和跨基地复制推广。

## 风险与应对

- CSV 题库缺少知识模块：MVP 默认未分类，二期补充模块字段和批量编辑。
- 题目质量不稳定：后台保留题目编辑和停用能力，导入后管理员可修正。
- LLM 成本和质量风险：AI 讲解按需触发并缓存，默认使用 CSV 原解析作为稳定讲解。
- 飞书接入不确定：MVP 使用工号或手机号登录，数据库预留飞书用户 ID。
- 排行榜口径争议：全局榜和题库榜并存，积分规则后台可配置。
- 云服务器安全：使用 HTTPS、管理员密码哈希、API 鉴权和数据库备份。
