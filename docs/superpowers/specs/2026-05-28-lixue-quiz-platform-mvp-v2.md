## 背景

项目书《「砺学」AI知识教育刷题系统项目申报书》提出建设飞书内 H5 刷题工具，用于员工碎片化练习、错题巩固、AI 讲解、积分排名和培训数据汇总。

结合当前已准备的 `question/questions.csv` 题库内容，第一版产品定位为“砺学多题库学习刷题平台”。MVP 先上线设备/岗位知识题库，完成真实内测闭环；飞书登录、飞书通知、LLM 自动出题、多基地权限和复杂运营活动放入后续阶段。

当前题库样本：

- 总题数：169。
- 单选题：112。
- 多选题：31。
- 判断题：26。
- 题库字段：题号、题型、题干、选项A、选项B、选项C、选项D、选项E、正确答案、正确答案文本、解析。

## 产品定位

砺学是面向基地员工的多题库刷题平台。员工在移动端完成题库选择、练习答题、错题重练、模拟考试和排行榜查看。管理员在 Web 后台维护员工、题库、题目、积分规则、LLM 配置，并查看和导出学习数据。

MVP 的第一目标不是覆盖所有培训运营场景，而是验证：

- 题库能稳定导入和维护。
- 员工能顺利登录并完成练习。
- 答题、错题、积分和排行榜口径可被解释。
- 后台能导出基本学习数据。
- AI 讲解可以按需启用，失败时不影响核心刷题流程。

## MVP 目标

MVP 在 1-2 个月内完成可真实内测的版本，部署到云服务器，支撑员工登录、多题库选择、练习答题、错题库、基础排行榜、轻量模拟考试、按需 AI 讲解、管理后台导入、数据看板和基础导出。

MVP 内测验收标准：

- 可成功导入当前 `question/questions.csv` 的 169 道题，并展示导入预检结果。
- 至少支持 30-50 名员工账号导入和登录。
- 员工可完成单选、多选、判断题练习。
- 答题提交后可看到正确答案、原始解析和错题状态。
- 错题可重新作答，答对后标记为已掌握。
- 全局排行榜和题库排行榜按已定义积分规则更新。
- 管理员可导出用户学习汇总、答题记录和模拟考试成绩。
- LLM 未配置或调用失败时，员工仍可使用原始解析完成学习。
- 云服务器重启后数据不丢失。

## 技术架构

- 前端：React + Vite，包含员工 H5 和管理后台。
- 后端：FastAPI，提供认证、题库、题目、练习、错题、积分、考试、排行榜、AI 讲解、导入导出和看板 API。
- 数据库：PostgreSQL。
- 部署：云服务器 + Nginx + HTTPS 域名。
- LLM：OpenAI-compatible 配置，支持 `base_url`、`api_key`、`model`。
- 飞书：MVP 只预留字段，不显示真实飞书登录入口。

## 用户与权限

### 员工用户

员工通过后台导入生成。员工使用工号或手机号加密码登录。

员工字段：

- 姓名。
- 工号。
- 手机号。
- 部门。
- 状态：启用、停用。
- 预留飞书用户 ID。
- 密码哈希。
- 创建时间和更新时间。

账号规则：

- 工号或手机号至少填写一个。
- 工号在系统内唯一。
- 手机号在系统内唯一。
- 导入时可设置统一初始密码，也可由系统生成随机初始密码。
- 管理员可重置员工密码。
- 员工停用后禁止登录；已有访问 token 应在服务端校验用户状态后失效。

### 管理员

MVP 后台只有管理员一种角色。管理员可使用全部后台功能，包括员工导入、题库导入、题目维护、积分规则配置、LLM 配置、数据看板和报表导出。

管理员账号要求：

- 密码必须哈希保存。
- 生产环境禁止使用默认密码。
- 管理员登录失败需要限速或记录失败次数。
- 关键操作需要写入审计日志。

## 员工端功能

### 登录

员工输入工号或手机号、密码登录。登录成功后进入题库选择页。登录失败时显示明确但不过度暴露信息的错误提示，例如“账号或密码错误”。

### 题库选择

员工登录后查看可用题库。每个题库显示：

- 题库名称。
- 题目数。
- 已练习题数。
- 个人正确率。
- 个人题库积分。
- 是否有可用模拟考试。

停用题库不向员工展示，但历史答题和考试记录仍可在后台保留。

### 练习答题

MVP 支持按题库随机练习。题型包括：

- 单选题：只能选择一个选项。
- 多选题：可选择多个选项，提交前按选项字母排序归一化。
- 判断题：按 CSV 内容映射为判断选项，前端可展示为“正确/错误”或原始选项文本。

答题提交后立即展示：

- 是否正确。
- 正确答案。
- 正确答案文本。
- CSV 原始解析。
- 是否进入错题库。
- 本次获得积分。

答题记录必须持久化，包含用户、题库、题目、选择答案、是否正确、来源、得分和提交时间。

### 错题库

员工答错题目后，题目进入个人错题库。

错题状态：

- `open`：未掌握。
- `mastered`：已掌握。

规则：

- 同一用户同一题只保留一条错题状态记录。
- 重复答错时增加错误次数并更新最后错误时间。
- 错题重做正确后标记为已掌握。
- 已掌握题再次答错时重新打开为未掌握。

### AI 讲解

答题反馈页和错题详情页默认展示 CSV 原始解析。员工点击“AI 讲解”时，后端调用 OpenAI-compatible LLM 生成补充讲解。

AI 讲解内容包括：

- 正确答案说明。
- 易错点。
- 关联知识提示。
- 简短学习建议。

AI 讲解规则：

- LLM 未启用时，前端隐藏或禁用入口。
- LLM 调用失败时，前端保留原始解析并提示稍后再试。
- AI 内容标识为辅助讲解，不覆盖标准答案。
- 生成结果写入缓存。
- 缓存键至少包含题目 ID、题目更新时间、模型、prompt 版本。
- 需要记录调用状态、耗时、错误原因和估算成本，便于排查和控费。

### 排行榜

MVP 提供：

- 全局总榜：按所有题库有效积分合计排序。
- 题库榜：按指定题库有效积分排序。

排序规则：

- 首先按积分降序。
- 同分时按正确率降序。
- 再同分时按累计答题数降序。
- 再同分时按最近得分时间升序。

排行榜显示字段：

- 排名。
- 姓名。
- 部门。
- 积分。
- 正确率。

### 模拟考试

MVP 提供轻量模拟考试。管理员可为题库配置默认考试题量和限时。

考试规则：

- 开始考试时服务端随机抽题，并保存本次考试题目快照。
- 考试题目快照不因后续题目编辑、停用或删除而改变。
- 每次考试只能提交一次。
- 服务端根据开始时间和限时判断是否超时。
- 交卷后展示得分、正确率、正确数、总题数和错题明细。

### 个人中心

个人中心展示：

- 姓名。
- 部门。
- 总积分。
- 累计答题数。
- 正确率。
- 未掌握错题数量。
- 考试记录入口。

## 管理后台功能

### 管理员登录

后台使用管理员账号密码登录。MVP 不做多角色权限，但所有后台接口必须校验管理员身份。

### 员工导入

管理员通过 CSV 或 Excel 导入员工信息。

导入字段：

- 姓名。
- 工号。
- 手机号。
- 部门。
- 状态。
- 初始密码，可选。

导入要求：

- 支持导入预检。
- 展示总行数、可导入行数、错误行数。
- 行级错误必须包含行号和错误原因。
- 工号或手机号重复时必须报告。
- 导入提交需要生成导入批次记录。

### 题库管理

管理员可创建、编辑、启用和停用题库。

题库字段：

- 名称。
- 说明。
- 状态。
- 默认考试题量。
- 默认考试限时。
- 创建时间。
- 更新时间。

### 题目导入

管理员选择目标题库后导入 CSV 题目。

MVP 支持当前 `question/questions.csv` 字段：

- 题号。
- 题型。
- 题干。
- 选项A。
- 选项B。
- 选项C。
- 选项D。
- 选项E。
- 正确答案。
- 正确答案文本。
- 解析。

题型映射：

- `单选题` -> `single`。
- `多选题` -> `multiple`。
- `判断题` -> `judgment`。

导入预检规则：

- 必须有题号、题型、题干和正确答案。
- 题型必须在支持范围内。
- 单选题答案只能有一个选项。
- 多选题答案可包含多个选项，系统归一化为按字母排序的格式。
- 正确答案必须能在已有选项中找到。
- 空选项不参与答案校验。
- 同一题库内题号重复必须报告。
- 缺少解析不阻止导入，但需要提示。

当前 CSV 不包含知识模块和难度。MVP 导入时将知识模块默认设为“未分类”，难度默认设为“普通”。后续可通过后台编辑或二期批量补充。

### 题目维护

管理员可查看、搜索、编辑、启用和停用题目。

题目停用后：

- 不再进入练习抽题。
- 不再进入新考试抽题。
- 不影响历史答题记录和历史考试记录。

题目编辑后：

- 更新题目更新时间。
- 使对应 AI 讲解缓存失效。
- 历史答题记录保留当时答题结果。

### 积分规则配置

后台支持配置：

- 答题基础分。
- 答对额外分。
- 错题重做答对分。
- 完成模拟考试分。
- 每日积分上限。

积分规则：

- 积分变动必须写入积分流水。
- 积分规则变更只影响后续行为，不回算历史积分。
- 同一用户同一题在同一天重复练习，基础分最多获得一次。
- 答错不给答对额外分。
- 每日积分上限按用户全局计算。
- 管理员可查看积分流水，但 MVP 不提供手工改分功能。

### 数据看板

后台看板展示：

- 参与人数。
- 活跃人数。
- 累计答题数。
- 整体正确率。
- 题库正确率。
- 未掌握错题数量。
- 高频错题。
- 模拟考试成绩分布。
- 排行榜概览。

### 报表导出

后台支持导出：

- 用户学习汇总。
- 答题记录。
- 模拟考试成绩。

导出要求：

- 支持 CSV，Excel 可作为增强能力。
- 导出操作写入审计日志。
- 大批量导出可异步处理，MVP 可先同步导出。

### LLM 配置

后台支持配置：

- `base_url`。
- `api_key`。
- `model`。
- 启用状态。
- prompt 版本。

安全要求：

- `api_key` 不在前端明文展示。
- 保存后的 key 默认掩码显示。
- 生产环境应使用环境变量或加密存储。
- 配置变更写入审计日志。

## API 约定

- 普通响应使用 JSON。
- 登录后使用 `Authorization: Bearer <token>`。
- 员工接口前缀：`/api`。
- 管理后台接口前缀：`/api/admin`。
- 导入接口需要区分预检和提交。
- 导出接口返回文件流或导出任务状态。

## 安全与合规

MVP 必须满足：

- 全站 HTTPS。
- 管理员和员工密码哈希存储。
- 生产环境替换默认管理员密码。
- 生产环境替换默认 JWT secret。
- 后台接口鉴权。
- 员工接口鉴权。
- 停用用户无法继续访问。
- LLM API key 不返回前端。
- 数据库定期备份。
- 关键后台操作写入审计日志。

生产部署前检查项：

- `JWT_SECRET` 已更换。
- `ADMIN_PASSWORD` 已更换。
- PostgreSQL 密码已更换。
- HTTPS 域名已配置。
- 数据库备份命令已记录。
- 数据库恢复流程至少验证一次。
- LLM API key 仅保存在环境变量或加密字段中。
- 飞书集成保持关闭。

## 分阶段路线

### MVP

## 风险与应对

- CSV 题库质量不稳定：通过导入预检、行级错误报告、题目编辑和停用降低风险。
- 题型解析不一致：在导入阶段统一题型映射和答案归一化。
- 积分口径争议：通过积分流水、每日上限和明确排序规则保持可解释。
- LLM 成本和质量风险：AI 讲解按需触发、缓存复用、失败降级到原始解析。
- 飞书接入不确定：MVP 不依赖飞书，仅预留用户 ID 字段。
- 云服务器安全风险：使用 HTTPS、密码哈希、API 鉴权、审计日志和数据库备份。
- 范围膨胀风险：按 P0、P1、P2 分层交付，优先保证可内测闭环。
